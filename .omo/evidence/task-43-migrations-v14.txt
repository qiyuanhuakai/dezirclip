PRE-EXISTING BUG FIXES REQUIRED FOR TEST VERIFICATION
-----------------------------------------------------
The following two one-line changes were required to enable `cargo test`
to compile and run. They are NOT new functionality for the v14 task but
are bundled into the commit because `cargo test` is the verification tool
mandated by the task.

1. src-tauri/src/services/ocr/linux.rs:
   Added `#[derive(Debug)]` to OcrService.
   Reason: pre-existing test in linux.rs uses `format!("{:?}", svc)` but
   OcrService had no Debug impl. Introduced in commit 5484fed (OCR
   Windows service). This blocks ALL test compilation in the binary.

2. src-tauri/src/database.rs:
   setup_test_db() schema now includes ocr_text TEXT and ocr_status TEXT
   NOT NULL DEFAULT 'pending' columns.
   Reason: clipboard_repo.rs's INSERT now writes these two columns. The
   integration test database::tests::test_save_and_get_history uses a
   minimal manual schema (not run_migrations), so the schema must mirror
   the production columns.

3. src-tauri/src/infrastructure/repository/clipboard_repo.rs:
   Updated test_v13_adds_content_kinds_column to delete v14 marker too.
   Reason: with v14 added, current_version=14 after the v13 marker is
   deleted, so `current_version < 13` is false and v13 won't re-apply.
   Test was passing before v14 only because there was no v14 to mask
   v13's version-13 condition. v14 is now also reset in the test.
